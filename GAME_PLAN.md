# せなくまゲームプロジェクト 大枠プラン

YouTuber「せなくま」さん（[せなくまチャンネル](https://www.youtube.com/@senakumadesu)）を主人公とした、**2Dゲームと3Dゲームの2本立て**のファンゲームプロジェクトの全体計画書。

せなくまさんは、あつまれ どうぶつの森・キングダムハーツ・マリオシリーズなどを中心に活動するゲーム実況者で、癒し系の声とゆったりした実況、かわいい雰囲気のチャンネルアートが特徴です。ゲームのトーン＆マナーもこれに合わせ、**「かわいい・ゆるい・癒し系」** を軸にします（🍟ポテトをモチーフとして活用）。

---

## 1. プロジェクト全体像

| 項目 | 内容 |
|---|---|
| 成果物 | ① 2Dミニゲーム ② 3D箱庭ゲーム ③ 両者をまとめるWebポータル |
| プラットフォーム | Webブラウザ（PC・スマホ両対応）。GitHub Pagesで無料配信 |
| 対象プレイヤー | せなくまさんのファン・視聴者。ゲーム初心者でも遊べる難易度 |
| 開発言語 | TypeScript（ゲーム本体）＋ Python（素材加工ツール） |

ブラウザ配信を推奨する理由：インストール不要でファンがリンク1つで遊べる、このリポジトリだけで開発から公開まで完結する、2Dと3Dを同じ技術基盤（TypeScript + Vite）で作れる、の3点です。

## 2. ⚠️ 最初に必ず確認：素材の権利について

YouTubeのサムネイルからキャラクターを切り抜いて使う計画のため、**着手前にここをクリアにすることが最優先タスク**です。

- サムネイルやキャラクターイラストの著作権は、せなくまさん本人および担当イラストレーターに帰属します。
- 本人・公認の企画であれば問題ありませんが、そうでない場合は **せなくまさん本人（および必要ならイラストレーター）の許諾を得る**、またはチャンネルの二次創作ガイドラインを確認することが必須です。
- 公開範囲（限定公開か一般公開か）、収益化の有無（**非収益・ファンメイド明記を推奨**）も合わせて決めます。
- 許諾が得られない場合の代替案：サムネイルの直接使用はやめ、**オリジナルのファンアート（自作イラスト）をキャラクター素材にする**方針に切り替えます（以降のプランはそのまま流用可能）。

`assets/raw/` に素材を保存する際は、出典URL・取得日・許諾状況をメモしたテキストを必ず添付する運用にします。

## 3. キャラクター素材の制作パイプライン

### 3-A. 2D素材（サムネイル切り抜き → スプライト化）

```
サムネイル収集 → 高解像度化 → 背景除去 → パーツ整理 → スプライトシート化
```

1. **収集**: YouTube Data API または `https://i.ytimg.com/vi/<動画ID>/maxresdefault.jpg` でサムネイルを取得。キャラクターが大きく写り、表情・ポーズのバリエーションがあるものを10〜20枚選定。
2. **高解像度化**: Real-ESRGAN（アニメ・イラスト用モデル）で2〜4倍にアップスケール。
3. **背景除去**: `rembg`（自動）＋ Segment Anything（細部の指定切り抜き）で人物のみ抽出。仕上げは手動レタッチ。
4. **バリエーション作成**: 表情差分（通常・笑顔・驚き）、向き（左右反転）を整理。歩行などのアニメーションは、切り抜き1枚を「ゆらし・バウンド」させる**ゆるアニメ方式**で表現（切り抜き素材と相性が良く、チャンネルの雰囲気にも合う）。
5. **スプライトシート化**: TexturePacker または自作Pythonスクリプトでゲームエンジン用にパッキング。

### 3-B. 3Dモデル（参考画像 → 3Dキャラクター）

切り抜いた2D画像は3Dモデルの**参考資料（リファレンス）**として使い、モデル自体は新規制作します。

- **本命ルート**: **VRoid Studio** でアニメ調3Dモデルを制作 → VRM形式で出力。無料・アニメ調に強い・Three.jsに `three-vrm` でそのまま読み込める。
- **こだわりルート**: Blenderでフルスクラッチ（デフォルメ・ちびキャラ調にしたい場合）。
- **下書き補助**: 画像→3D生成AI（TripoSRなど）でラフを作り、Blenderで整える方法も検討可。
- **アニメーション**: VRM標準ボーン＋Mixamoのモーション（歩く・走る・ジャンプ・手を振る）を流用。

## 4. ゲーム企画案

### ① 2D「せなくまラン 🍟」（横スクロールアクション）

- せなくまさんが走ってポテトを集め、障害物をジャンプで避けるエンドレスランナー。
- スコア＝集めたポテト数。ハイスコアをローカル保存。
- 操作はタップ／スペースキーのみ。配信の待機画面やファンの暇つぶしにぴったりの手軽さ。
- 拡張案: ステージテーマを実況ゲーム（森・お城・キノコ王国風）のオマージュにする。ミニゲーム（もぐらたたき風、記憶ゲームなど）を追加してミニゲーム集化。

### ② 3D「せなくまアイランド 🏝️」（箱庭探索アドベンチャー）

- あつ森実況へのオマージュ。小さな島をせなくまさん（3Dモデル）で自由に歩き回る。
- 島に隠されたポテトやチャンネルゆかりのアイテムを集めるコレクション要素。
- 住民キャラに話しかけるとチャンネルの名言・小ネタが出る。
- 操作: WASD／バーチャルパッド＋カメラ回転。戦闘なしの完全癒し系。

### ③ Webポータル

- 2ゲームへの入口となるランディングページ。チャンネルリンク、クレジット（イラスト出典・許諾表記）を掲載。

## 5. 技術スタック

| 領域 | 採用技術 | 理由 |
|---|---|---|
| 2Dゲーム | **Phaser 3** + TypeScript | 2Dブラウザゲームの定番。スプライトシート・物理・シーン管理が揃っている |
| 3Dゲーム | **Three.js** + `three-vrm` + TypeScript | VRMモデルを直接読み込める。ブラウザ3Dの標準 |
| ビルド | Vite（モノレポ構成） | 2D/3D/ポータルを1リポジトリで管理、開発サーバーが高速 |
| 素材加工 | Python（rembg, Real-ESRGAN, Pillow） | 切り抜き・アップスケール・スプライトシート生成の自動化 |
| 3D制作 | VRoid Studio ＋ Blender ＋ Mixamo | 無料でアニメ調モデル→リグ→モーションまで完結 |
| 配信 | GitHub Pages ＋ GitHub Actions | push時に自動ビルド・自動デプロイ |

> 代替案: UnityやGodotでの開発も可能（スマホアプリ化したい場合はUnityが有力）。ただし配布のしやすさ・開発の身軽さからまずはWeb版を推奨。

## 7. 開発ロードマップ

| フェーズ | 内容 | 完了条件（マイルストーン） |
|---|---|---|
| **Phase 0** 権利確認 | 許諾取得 or ガイドライン確認、公開方針決定 | 素材使用の可否が文書で確定 |
| **Phase 1** 素材パイプライン | サムネイル収集・切り抜きツール整備、スプライト第1弾 | せなくまさんの立ち絵スプライト3種以上完成 |
| **Phase 2** 2D MVP | Phaserでランナーのコア（走る・跳ぶ・ポテト・スコア） | ブラウザで一通り遊べる |
| **Phase 3** 2D仕上げ＋公開 | 音・タイトル画面・スマホ対応、GitHub Pages公開 | 公開URLでプレイ可能 🎉 |
| **Phase 4** 3Dモデル制作 | VRoidでモデル制作、Mixamoモーション適用 | ブラウザ上でモデルが歩く |
| **Phase 5** 3D MVP | 島マップ・移動・カメラ・アイテム収集 | 島を探索して収集が完結する |
| **Phase 6** 3D仕上げ＋統合公開 | 住民会話・演出・ポータル統合 | ポータルから両ゲームが遊べる |

進め方のポイント: **2Dを先に完成・公開**して手応え（ファンの反応）を得てから3Dに投資する順番にしています。Phase 2完了時点で最初の「遊べるもの」ができます。

## 8. リスクと対応

| リスク | 対応 |
|---|---|
| 素材の許諾が得られない | 自作ファンアートに切り替え（パイプラインはそのまま使える） |
| 切り抜き品質が低い（サムネイルの解像度不足） | Real-ESRGANでの補完＋トレス（描き起こし）併用 |
| 3D制作の工数が想定超過 | VRoidの既成テンプレートを最大活用。3Dは2D公開後に着手する計画なので中断も可能 |
| スマホでの3Dパフォーマンス | ローポリ・ライトマップ焼き込みで軽量化。品質設定の切り替えを用意 |
